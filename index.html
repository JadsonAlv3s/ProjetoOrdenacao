Introdução

O que é Ordenação?

Ordenação é o processo de reorganizar elementos de uma coleção em uma sequência lógica — crescente ou decrescente — otimizando operações subsequentes.

  • 🔍 Viabiliza busca binária eficiente
  • 📈 Otimiza algoritmos dependentes
  • 🗂️ Organiza dados para visualização
  • ⚙️ Fundamental em bancos de dados
💡 Por que esses dois?

O QuickSort é o algoritmo mais usado na prática por sua eficiência média. O SmoothSort é uma variante elegante do HeapSort que se adapta a dados parcialmente ordenados, atingindo O(n) no melhor caso.

O(n log n) Caso médio
O(1) Espaço (SmoothSort)
História

A origem do
QuickSort

O QuickSort foi desenvolvido em 1959 pelo cientista da computação britânico Tony Hoare, enquanto trabalhava em um projeto de tradução automática na Universidade de Moscou.

Hoare precisava ordenar palavras para um dicionário russo-inglês e, ao buscar uma solução eficiente, concebeu a ideia de particionar os dados em torno de um pivô. O algoritmo foi publicado formalmente em 1961.

Desde então, tornou-se o algoritmo de ordenação mais utilizado na prática, sendo a base da função sort() em linguagens como C, C++ e Java.

1959

Concepção

Tony Hoare desenvolve o algoritmo durante estágio na Universidade de Moscou, trabalhando com tradução automática.

1961

Publicação

O artigo "Algorithm 64: Quicksort" é publicado na Communications of the ACM, formalizando o algoritmo.

Hoare publica "Quicksort" no Computer Journal com análise completa de complexidade e provas de correção.

1970+

Otimizações

Sedgewick, Bentley e outros propõem melhorias: mediana de três, Introsort, e versões híbridas usadas até hoje.

🏆 Legado

Tony Hoare recebeu o Prêmio Turing em 1980 por suas contribuições à ciência da computação, incluindo o QuickSort e a lógica de Hoare para verificação de programas.

QuickSort

Como funciona o
QuickSort?

Utiliza a estratégia dividir para conquistar: escolhe um pivô, particiona o array em elementos menores e maiores que o pivô, e aplica recursivamente o mesmo processo.

1 Escolhe um elemento como pivô
2 Particiona: menores à esquerda, maiores à direita
3 Aplica recursivamente nas sub-partições
Melhor caso: O(n log n)
Caso médio: O(n log n)
Pior caso: O(n²)
Espaço: O(log n)
quicksort.py 
def quicksort(arr, low, high):
    if low < high:
        pi = partition(arr, low, high)
        quicksort(arr, low, pi - 1)
        quicksort(arr, pi + 1, high)

def partition(arr, low, high):
    pivot = arr[high]  # pivô
    i = low - 1
    for j in range(low, high):
        if arr[j] <= pivot:
            i += 1
            arr[i], arr[j] = arr[j], arr[i]
    arr[i+1], arr[high] = arr[high], arr[i+1]
    return i + 1
⚠️ Atenção

O pior caso O(n²) ocorre quando o pivô é sempre o menor ou maior elemento. Soluções: pivô aleatório ou mediana de três.

História

A origem do
SmoothSort

O SmoothSort foi inventado em 1981 pelo lendário cientista da computação holandês Edsger W. Dijkstra, um dos mais influentes pensadores da área.

Dijkstra buscava criar um algoritmo que combinasse as garantias do HeapSort (pior caso O(n log n)) com a capacidade de se adaptar a dados já parcialmente ordenados — algo que o HeapSort tradicional não faz.

A solução foi usar os números de Leonardo como base para uma estrutura de heap implícita, resultando em um algoritmo que "suaviza" seu comportamento conforme a desordem dos dados — daí o nome SmoothSort.

1981

Publicação original

Dijkstra publica o manuscrito EWD796 "Smoothsort, an alternative for sorting in situ", descrevendo o algoritmo em detalhes.

1981

Números de Leonardo

A escolha dos números de Leonardo (em vez de potências de 2) permite que heaps adjacentes se combinem de forma natural.

1984

Análise de adaptabilidade

Estudos confirmam que o SmoothSort atinge O(n) em sequências já ordenadas, validando a proposta de Dijkstra.

2000+

Redescoberta

Implementações modernas em C++, Rust e Java surgem, e o algoritmo ganha atenção em competições e pesquisas sobre ordenação adaptativa.

🧠 Sobre Dijkstra

Edsger Dijkstra (1930–2002) é conhecido pelo algoritmo de caminho mínimo, pela programação estruturada e pelo famoso artigo "Go To Statement Considered Harmful". Recebeu o Prêmio Turing em 1972.

SmoothSort

Como funciona o
SmoothSort?

Criado por Edsger Dijkstra (1981), é uma variante adaptativa do HeapSort. Usa uma floresta de heaps de Leonardo para explorar a ordem pré-existente nos dados.

1 Constrói uma floresta de heaps de Leonardo
2 Mantém a propriedade de heap entre as raízes
3 Extrai o máximo e reconstrói a estrutura
Melhor caso: O(n)
Caso médio: O(n log n)
Pior caso: O(n log n)
Espaço: O(1)
🔢 Números de Leonardo

Sequência similar a Fibonacci:
1, 1, 3, 5, 9, 15, 25, 41, 67...

L(n) = L(n-1) + L(n-2) + 1

Cada heap na floresta tem tamanho igual a um número de Leonardo.

✨ Vantagem adaptativa

Se os dados já estão parcialmente ordenados, o SmoothSort detecta isso e reduz o trabalho, aproximando-se de O(n).

Conclusão

Quando usar
cada um?

⚡ QuickSort

  • Dados completamente aleatórios
  • Quando performance média importa mais
  • Implementação simples é prioridade
  • Uso geral (padrão em muitas linguagens)

🌊 SmoothSort

  • Dados parcialmente ordenados
  • Garantia de O(n log n) é necessária
  • Memória extra não é aceitável
  • Sistemas embarcados / tempo real

O QuickSort domina na prática pela simplicidade e cache-friendliness. O SmoothSort é a escolha teórica superior quando adaptabilidade e garantias de pior caso são essenciais.
